闭包

Posté le 2016-12-17

闭包（closure）是Javascript语言的一个难点，也是它的特色，很多高级应用都要依靠闭包实现。

闭包的特性

闭包有三个特性

  • 函数的嵌套
  • 函数内部可以引用外部的参数和变量
  • 参数和变量不会被垃圾回收机制回收

闭包的定义及其优缺点

闭包是指有权限访问另一个函数作用域中的变量的函数，创建闭包的最常见方式就是在一个函数内创建另一个函数，函数通过另一个函数访问这个函数的局部变量
闭包的缺点：就是常驻内存，会增大内存使用量，使用不当很容易造成内泄露

闭包是js的一大特点，主要应用闭包场合主要是为了：设计私有变量和方法

一般函数执行完之后，局部活动对象就会被销毁，内存中仅仅保存全局作用域，但闭包的情况不同。闭包会使变量始终保存在内存中，如果使用不当会增大内存消耗

学习之初

Vue.js是什么为什么要学习Vue

  • Vue.js（读音 /vjuː/, 类似于 view） 是一套构建用户界面的 渐进式框架。
  • vue的设计模式采用：自底向上增量开发的设计模式。
  • Vue的核心库只关注视图层
  • 简单容易上手，容易与其他库或已有项目整合
  • 采用单文件组件和vue生态系统支持的库开发的复杂单页应用

Vue.js的目标是通过简单地API实现相应的数据绑定和组合的视图组件

Vue的安装

兼容性

Vue.js 不支持 IE8 及其以下版本，因为 Vue.js 使用了 IE8 不能模拟的 ECMAScript 5 特性。 Vue.js 支持所有兼容 ECMAScript 5 的浏览器。

独立版本

直接下载并用 script标签引入，Vue 会被注册为一个全局变量。
重要提示：在开发时请用开发版本，遇到常见错误它会给出友好的警告。

NPM

在用 Vue.js 构建大型应用时推荐使用 NPM 安装， NPM 能很好地和诸如 Webpack 或 Browserify 模块打包器配合使用。 Vue.js 也提供配套工具来开发单文件组件。
最新稳定版

在底层的实现上， Vue 将模板编译成虚拟 DOM 渲染函数。结合响应系统，在应用状态改变时， Vue 能够智能地计算出重新渲染组件的最小代价并应用到 DOM 操作上。

如果你熟悉虚拟 DOM 并且偏爱 JavaScript 的原始力量，你也可以不用模板，直接写渲染（render）函数，使用可选的 JSX 语法。

JavaScript之BOM

Posté le 2016-12-14

ECMAScript 是 JavaScript 的核心，但如果要在 Web 中使用 JavaScript，那么 BOM（浏览器对象模型）则无疑才是真正的核心。BOM 提供了很多对象，用于访问浏览器的功能，这些功能与任何网页内容无关。多年来，缺少事实上的规范导致 BOM 有很多问题，因为浏览器提供商会按照各自的想法随意去扩展它。W3C 为了把浏览器中 JavaScript 最基本的部分标准化，已经将 BOM 的主要方面纳入了 HTML5 的规范中。

window 对象

BOM 的核心对象是 window，它表示浏览器的一个实例。在浏览器中，window 对象有双重角色，它既是通过 JavaScript 访问浏览器窗口的一个接口，又是 ECMAScript 规定的 Global 对象。这意味着在网页中定义的任何一个对象、变量和函数，都以 window 作为其 Global 对象，因此有权访问 isNaN()、isFinite()、parseInt()、parseFloat() 等方法。
